# Lumina 课程数据：系统语言释义编写规范（HSK 3.0 等）

本规范适用于课程 JSON 中面向学习者的 **系统界面语言** 释义字段，包括但不限于：

- `dialogueCards[].lines[].translation`
- `grammar[].examples[].translation`、扩展区 `translations`、练习 `prompt` / `explanation` 等 **多语展示文案**

**不适用于**：课文汉字、拼音、以中文为主的教学讲解正文（`zh` / `cn` 固定为中文教材内容）。

---

## 核心原则

1. **禁止以「逐字对齐中文」为最终目标**  
   释义是「这个意思在目标语言里怎么说最自然」，不是字对字翻译。

2. **禁止机械拼接**  
   避免把中文语序、成分原样搬进韩语/日语/英语。

3. **优先母语者/教材体**  
   - 韩语：像韩国课堂里老师会用的、顺口、好懂的口语或教材体。  
   - 日语：像日本教科书或日常自然说法，避免「中文直译腔」。  
   - 英语：像真实教材或自然对话，简洁、地道，符合英语问答习惯。

4. **意思准确优先于形式一致**  
   允许因语境省略、合并、换句式；短答（如「喜欢。」）在英语里用 *Yes, I do.* / *I do.* 等符合上下文的形式。

5. **全课风格统一**  
   同一课程系列中，同一语言下称谓、礼貌级别、标点与缩写习惯应前后一致；新增课不得明显低于前几课的人工润色标准。

---

## 验收检查（新课必做）

- [ ] **kr**：朗读是否像韩国人自然会说的话，而非翻译腔  
- [ ] **jp**：是否符合日语语感和教材体，无中文硬套  
- [ ] **en**：是否像英语母语教材句子，问答逻辑正确  
- [ ] 与 **第 1–7 课已审定风格** 对比，无明显质量落差  

---

## 数据层要求

- 自然释义 **写入课程 JSON 对应字段**（如 `translation.kr` / `jp` / `en`），**不要**依赖前端临时替换。  
- 若曾用脚本批量生成初稿，**定稿前必须人工按本规范重写**。
