## 1. 四语自然度总原则（与 Chief Agent 对齐）

| 语言 | 要求 |
|------|------|
| **cn** | 适合中文学习平台，自然、清晰，避免英译腔 |
| **kr** | 自然、亲切，符合韩国学习者阅读习惯与敬语习惯 |
| **en** | 简洁、UI 友好，适合国际用户 |
| **jp** | 自然、礼貌（です・ます 体系统一），避免中→日机械映射 |

不确定项一律进入 §6 **Pending Approval**，**禁止**静默覆盖既有译文。

## 2. 架构事实：双轨文案系统

| 轨道 | 路径 | 现状摘要 |
|------|------|----------|
| **A. 运行时主语言包** | `lang/{cn,kr,en,jp}.json` | 体量大、按 namespace 嵌套；**四语齐全**（结构一致性需 spot-check） |
| **B. 模块化 i18n（fetch）** | `ui/i18n/*.json` + `ui/i18n/index.js` | 设计为 `key → {cn,kr,en}`；`jsonToDict` **仅合并 kr/cn/en**，**无 jp 槽位** |
| **C. 旧版内联 DICT** | `ui/i18n.js` 内 `DICT.kr/cn/en/jp` | jp 仅覆盖少量 key（如 word_study_*），与 A/B 并存 |

**风险（warning 级）**: 若某屏同时混用 `t("nav_home")`（轨道 A）与 `loadAllI18n`（轨道 B），日语界面可能出现 **部分 key 回退为 key 字符串** 或 **英/韩混显**。需产品定义「单一真源」或统一加载顺序（Round 2+ 讨论，本报告不改代码）。

## 5. 表 3 · 四语自然度与一致性 **风险清单**（抽样 + 启发式）

> 下列 **不** 代表「必须修改」，仅列 **待评审**；是否采纳须人工 + 母语者。

| ID | 位置 | 现象 | 建议方向（草稿） | confidence |
|----|------|------|------------------|------------|
| R1 | `ui/i18n/index.js` | `jsonToDict` / `loadAllI18n` 无 `jp` | Round 2 提案：`out.jp` + `merged.jp`（**须代码审**） | medium |
| R2 | `ui/i18n/practice.json` → `practice_submit` | jp 为「提出」 | UI 场景亦常说「送信する」；是否统一为「解答を送信」类表述 | medium |
| R3 | `ui/i18n/practice.json` → `practice_listen` | jp 为「発音」而 en 为「Speak」 | en/jp 语义不对齐（听 vs 说）；或 en 文案历史有误 | **high 风险需确认** |
| R4 | `lang/cn.json` → `hanja.basic3000.levelDesc` | 含 `{count}` 占位 | 需与 `page.hanja.js` 中 `i18n.t("hanja.basic3000.levelDesc", { count })` 四语占位符一致 | medium |
| R5 | 双轨系统 | 同屏混用 key 命名风格 | 长期技术债；非单语问题 | low（架构） |
