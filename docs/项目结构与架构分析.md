# Lumina / Hanjiapass 项目结构与架构分析

## 一、项目根目录概览

```
hanjiapass/
├── index.html          # 单页入口
├── ui/                  # 前端 UI 与逻辑
│   ├── app.js           # 应用入口（路由注册、挂载导航/面板）
│   ├── router.js        # Hash 路由
│   ├── dataPaths.js     # 数据 URL 构建（vocab/lessons/strokes）
│   ├── i18n.js          # 多语言
│   ├── components/      # 导航、AI 面板、学习面板等
│   ├── core/            # 课程引擎、步骤运行器、生词步骤等
│   ├── modules/         # HSK、笔画等业务模块
│   ├── pages/           # 各路由对应页面
│   └── styles/          # 全局样式
├── data/                # 静态课程与资源数据
│   ├── vocab/           # 词汇 JSON（按版本 hsk2.0 / hsk3.0）
│   ├── lessons/         # 课程序列与单课详情 JSON
│   └── strokes/         # 汉字笔画 SVG（按 Unicode 码点命名）
└── tools/               # 脚本工具（如 clean_vocab.py）
```

## 二、index.html 如何启动网站

1. **唯一 HTML 入口**  
   整站是单页应用（SPA），仅有一个 `index.html`。  
   所有“页面”由 JS 根据 Hash 路由动态渲染到同一个 `<main id="app">` 里。

2. **加载顺序**  
   - **样式**：`base.css`、`panels.css`（带版本号做缓存控制）。  
   - **全局工具**：先加载 **`dataPaths.js`**（经典脚本），在 `window.DATA_PATHS` 上暴露 `vocabUrl`、`lessonsUrl`、`strokeUrl` 等，供后续任何模块使用。  
   - **应用入口**：再加载 **`app.js`**（`type="module"`），作为 ES 模块入口。

3. **DOM 结构**  
   - `#siteNav`：顶部导航挂载点，由 `app.js` 里 `mountNavBar(siteNav)` 填充。  
   - `#app`：主内容区，由 **router** 根据当前 hash 懒加载对应页面模块并挂载。  
   - `#siteFooter`：底部挂载点（若需可在此挂 footer 组件）。

4. **总结**  
   **index.html** 只做三件事：引入样式、引入 `dataPaths.js`、引入 `app.js`。  
   网站“启动”的真正逻辑在 **app.js**：初始化 i18n、挂载导航与全局面板、注册路由并启动 **router**。

## 三、router.js 如何控制页面

- **位置**：`ui/router.js`，由 `app.js` 以 ES 模块形式 `import { startRouter, registerRoute } from "./router.js"` 使用。

- **路由表**  
  使用 `Map` 存储：**hash → 动态 import 函数**。  
  在 `app.js` 中通过 `registerRoute("#home", () => import("./pages/page.home.js"))` 等形式注册；只保留第一段 hash（如 `#home/xxx` → `#home`），忽略 query。

- **启动**  
  `startRouter({ defaultHash: "#home", appId: "app", scrollTop: true })`：  
  - 若当前无 hash，设为 `defaultHash`。  
  - 监听 `hashchange`，并**在启动时用 `queueMicrotask` 立即执行一次渲染**，避免依赖“从空到有”的 hash 变化。  
  - 渲染目标为 `getElementById(appId)`（即 `#app`）。

- **一次导航流程**  
  1. 规范化 hash，若未注册则显示“页面不存在”并可选重试/回首页。  
  2. 显示“불러오는 중...”加载态，可选滚动到顶部。  
  3. 若存在上一页且导出 `unmount`，先执行 `unmount`。  
  4. 调用该路由对应的 **loader**：`await loader()` 得到页面模块。  
  5. 用统一约定调用页面：依次尝试 `mod.mount(ctx)`、`mod.init(ctx)`、`mod.default(ctx)`、`mod.render(ctx)`，任一个存在即执行；`ctx` 含 `root`（#app）、`hash`、`route`、`token`。  
  6. 出错则显示错误 UI（重试 / 回首页）。  
  “慢加载”仅做提示，不打断等待。

- **同 hash 不重复渲染**  
  若新 hash 与当前已渲染的 hash 相同且非强制刷新，且当前内容既非加载态也非空，则只发 `route` 事件（如给导航栏高亮用），不再重新加载模块。

- **强制刷新**  
  `navigateTo("#home", { force: true })` 可在 hash 不变时强制再执行一次上述渲染流程（例如首页“Home”点击刷新）。

- **总结**  
  **router.js** 是唯一的“页面控制器”：根据 hash 懒加载对应 `pages/page.*.js`，并把页面挂载/渲染到 `#app`；页面只需导出 `mount`/`init`/`default`/`render` 之一即可。

## 四、pages 目录的作用

- **职责**：每个文件对应一个路由（或子功能），负责该“页面”的 UI 和交互，由 router 按 hash 懒加载并挂载到 `#app`。

- **与路由的对应关系**（在 `app.js` 中注册）：

  | Hash        | 页面模块                 |
  |------------|--------------------------|
  | `#home`    | `page.home.js`           |
  | `#hsk`     | `page.hsk.js`            |
  | `#stroke`  | `page.stroke.js`         |
  | `#hanja`   | `page.hanja.js`          |
  | `#speaking`| `page.speaking.js`       |
  | `#travel`  | `page.travel.js`         |
  | `#culture` | `page.culture.js`        |
  | `#review`  | `page.review.js`         |
  | `#resources` | `page.resources.js`   |
  | `#teacher` | `page.teacher.js`        |
  | `#my`      | `page.my.js`             |
  | `#catalog` | `page.catalog.js`        |

- **约定**：每个页面模块需至少导出以下之一：`mount`、`init`、`default`、`render`，入参为 router 传入的 `ctx`（含 `root`、`hash`、`route` 等）。  
  例如 `page.home.js` 导出 `mount`/`render`/`default`，内部把首页 HTML 渲染到 `ctx.root`；`page.hsk.js` 导出 `mount`，内里拉取课表、渲染 HSK 布局并绑定事件。

- **子目录**：  
  - `pages/hsk/`：HSK 页专用（如 `state.js`、`modalMode.js`、`utils.js`），由 `page.hsk.js` 或相关模块引用。

- **总结**：**pages** 目录即“按路由拆分的页面模块”，router 只做“根据 hash 选哪个模块、往哪里挂载”，具体内容与数据加载由各 page 自己完成。

## 五、ui 模块结构

- **入口与路由**  
  - `app.js`：挂载 LESSON_ENGINE、lessonBridge、lessonStepRunner、wordPanel、i18n、导航栏、AI 面板、学习面板，注册所有路由并 `startRouter()`。  
  - `router.js`：见上文。  
  - `dataPaths.js`：基准路径与版本检测，提供 `DATA_PATHS.vocabUrl/lessonsUrl/strokeUrl` 等（先于 app.js 以普通脚本加载）。

- **components/**  
  全站复用：导航栏（`navBar.js`）、AI 面板（`aiPanel.js`）、学习面板（`learnPanel.js`）、词卡/弹窗（`wordPanel.js`）、footer、对话/弹窗基类等。

- **core/**  
  课程与步骤引擎：`lessonEngine.js`、`lessonBridge.js`、`lessonStepRunner.js`、`wordsStep.js`（生词步骤，会调 `DATA_PATHS.lessonsUrl` 或 `HSK_LOADER`）、`deriveLessonId.js`、`lang.js` 等。

- **modules/**  
  - **hsk**：`hskLoader.js`（经典脚本，挂 `window.HSK_LOADER`，提供 `loadVocab`/`loadLessons`/`loadLessonDetail`）、`hskLayout.js`、`hskRenderer.js`、`hskDeps.js`（可动态加载 hskLoader）、`hskHistory.js`、`hskUI.js`、以及 `ui/` 下的 hsk 子模块（hskData、viewLessonDetail、hskCache 等）。  
  - **stroke**：笔画播放与书写（`strokePlayer.main.js`、`strokePlayer.canvas.js`、模板与 trace 等），依赖 `DATA_PATHS.strokeUrl`。

- **pages/**  
  见上一节；页面内再按需引用 components、core、modules。

- **styles/**  
  `base.css`、`panels.css`、`hsk.css` 等全局样式。

- **其他**  
  `i18n.js`、`aiUI.js`、`learnPanel.js`、`hskLookup.js` 等；以及一些兼容用入口（如 `ui-stroke-player.js`、`ui-trace-canvas.js`）。

整体上，**ui** 是“单页前端的全部逻辑与界面”：入口 → 路由 → 页面与组件 → 业务模块（HSK/笔画）与核心引擎（课程/步骤），数据路径由 **dataPaths** 统一提供。

## 六、data 目录课程数据如何加载

- **目录约定**  
  - **词汇**：`data/vocab/<版本>/hsk<级别>.json`，如 `data/vocab/hsk2.0/hsk1.json`。  
  - **课程序列**：`data/lessons/<版本>/hsk<级别>_lessons.json` 或 `data/lessons/<版本>/hsk<级别>.json`（索引列表）。  
  - **单课详情**：`data/lessons/<版本>/<file>` 或回退到 `hsk<级别>_lesson<课号>.json`。  
  - **笔画**：`data/strokes/<Unicode 码点>.svg`，如 `23458.svg` 对应某个汉字。

- **路径由谁决定**  
  - **dataPaths.js**：根据 `window.__APP_BASE__` 或 `<base href>` 或 `location.pathname` 算出 BASE，再拼出 `vocabUrl(level)`、`lessonsUrl(level)`、`strokeUrl(char)` 等；可选 `localStorage.hsk_vocab_version`、版本号参数。  
  - **hskLoader.js**：内部也有一套 URL 规则，但会优先使用 `window.DATA_PATHS` 的 `vocabUrl`/`lessonsUrl`/`lessonDetailUrl`（若存在），再 fallback 到自己的路径；支持 404 时回退到 hsk2.0 等。

- **谁去请求数据**  
  - **HSK 列表与详情**：`page.hsk.js` 在 `mount` 后调 `loadLessons()`，内部用 `window.HSK_LOADER.loadLessons(state.lv, { version })`；点击某一课时用 `HSK_LOADER.loadLessonDetail(lv, lessonNo, { version, file })`，再配合 `HSK_LOADER.loadVocab` 得到该课生词。  
  - **HSK_LOADER** 由 **hskLoader.js** 以经典脚本形式挂到 `window`；若未预先在 index 或其它地方加载，需通过 **hskDeps.ensureHSKDeps()** 动态插入 `hskLoader.js`（当前代码中未见调用，实际部署时需在进入 #hsk 前或 index 中保证加载）。  
  - **课程步骤/生词**：`wordsStep.js` 等会先用 `HSK_LOADER.loadLesson(lessonId)`（若存在），否则用 `DATA_PATHS.lessonsUrl(lessonId)` 做 `fetch`，解析 pack 或单课 JSON 得到生词列表。  
  - **笔画**：笔画播放器通过 `DATA_PATHS.strokeUrl(字符)` 取 SVG 地址并加载。

- **版本与缓存**  
  - 版本：`localStorage.hsk_vocab_version`、或 `opts.version`、或默认 `hsk2.0`；hskLoader 内部有 30 分钟内存缓存（按 url/版本/级别等 key），并暴露 `setVersion`/`clearCache`。

- **总结**  
  **data** 目录是静态 JSON/SVG 资源；**加载流程**为：  
  **dataPaths 提供 URL → 页面或 core 模块用 HSK_LOADER / fetch 请求 → 解析后渲染或交给课程引擎/步骤器使用**。  
  课程数据流：`data/lessons`、`data/vocab` → **hskLoader / wordsStep** → **page.hsk / lessonEngine / wordPanel** 等。

## 七、整个网站架构流程图

下面用 Mermaid 画出从“打开 index.html”到“页面展示与数据加载”的流程。

```mermaid
flowchart TB
  subgraph 入口
    A[index.html] --> B[加载 dataPaths.js]
    B --> C[加载 app.js 模块]
  end

  subgraph app.js 初始化
    C --> D[挂载 LESSON_ENGINE / lessonBridge / stepRunner]
    D --> E[挂载 wordPanel / i18n]
    E --> F[mountNavBar #siteNav]
    F --> G[mountAIPanel / mountLearnPanel]
    G --> H[registerRoute 注册各 hash → page 模块]
    H --> I[startRouter defaultHash: #home, appId: app]
  end

  subgraph router 控制
    I --> J{监听 hashchange 或首次 queueMicrotask}
    J --> K[normalizeHash 取 # 第一段]
    K --> L{ROUTES.get(hash)}
    L -->|无| M[显示 404 / 重试]
    L -->|有| N[显示加载中]
    N --> O[loader = 动态 import 对应 page]
    O --> P[执行 page.mount/init/default/render(ctx)]
    P --> Q[内容渲染到 #app]
  end

  subgraph 页面与数据
    Q --> R[例如 #hsk → page.hsk]
    R --> S[getHSKLayoutHTML + bindEvents]
    S --> T[需要数据时]
    T --> U[window.HSK_LOADER.loadLessons / loadLessonDetail / loadVocab]
    U --> V[hskLoader 内部用 DATA_PATHS.vocabUrl/lessonsUrl 或自建 URL]
    V --> W[fetch data/vocab/... 或 data/lessons/...]
    W --> X[内存缓存可选]
    X --> Y[渲染课表/生词/对话等]
  end

  subgraph 数据路径
    dataPaths[dataPaths.js] --> vocabUrl[vocabUrl(level)]
    dataPaths --> lessonsUrl[lessonsUrl(level)]
    dataPaths --> strokeUrl[strokeUrl(char)]
    vocabUrl --> dataV[data/vocab/ver/hskN.json]
    lessonsUrl --> dataL[data/lessons/ver/hskN_lessons.json 等]
    strokeUrl --> dataS[data/strokes/码点.svg]
  end

  V -.-> dataPaths
```

## 八、简要总结表

| 项目       | 说明 |
|------------|------|
| **index.html** | 单页入口；只加载样式、dataPaths.js、app.js；#siteNav / #app / #siteFooter 三个挂载点。 |
| **router.js**  | Hash 路由；registerRoute 注册 hash → 动态 import；startRouter 监听 hash 并首次渲染；按 hash 懒加载 page 模块，调用 mount/init/default/render，目标 #app。 |
| **pages/**     | 各路由对应页面模块；导出 mount/init/default/render 之一；负责该页 UI 与数据请求（如 HSK 用 HSK_LOADER）。 |
| **ui 结构**    | app → router + components + core（课程/步骤引擎）+ modules（hsk、stroke）+ pages + styles；dataPaths 先于 app 提供数据 URL。 |
| **data 加载**  | dataPaths 提供 vocabUrl/lessonsUrl/strokeUrl；hskLoader 提供 loadVocab/loadLessons/loadLessonDetail（优先用 DATA_PATHS）；wordsStep 等用 loader 或 fetch 拉取 data 下 JSON；笔画用 strokeUrl 取 SVG。 |

以上即从“index 启动”到“router 控制页面”再到“pages 与 data 加载”的完整链路说明与架构图。
