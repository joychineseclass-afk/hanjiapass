# Lumina 页面导航规范

本文档说明 Lumina 全站**页面内信息架构**的约定，与顶端主导航（`#home`、`#exam`、`#resources` 等）配合使用。

## 1. 一级导航（仅顶端）

- 一级模块**只**放在顶端主导航栏。  
- 例如：首页、考试学习、儿童中文、会话、汉字学习、笔顺/跟写、文化、资料、教师工作区、我的学习。  
- 不要把「某页内部的分类」再堆到顶栏，避免与一级入口混淆。

## 2. 页面内分类：统一左侧

- 进入某个一级模块后，若该页有**多种并列内容/流程**（资料分类、教师子模块、考试子 Tab 等），**二级及以下导航应放在当前页的左侧**（或等同于「侧栏」的纵向导航区域）。  
- **不要**把新模块的入口直接**平铺成主区大卡片/大按钮列表**作为唯一导航方式。主区用于展示**当前选中的那一条**内容。  
- 实现上可复用与教师工作台一致的模式：如 `.teacher-shell` + 侧栏链接；全站通用壳见 `ui/styles/base.css` 中的 **`.page-shell`、`.section-side-nav`、`.section-main-panel`** 等 class（资料页等已使用）。

## 3. 层级与字号、缩进

- **二级导航**：在左侧，正常字重/字号（如 `.section-side-nav-item.level-2` 或与 `.teacher-shell-nav-link` 一致）。  
- **三级导航**：放在对应二级**下方**，字号略小、**左侧增加缩进**（如 `.section-side-nav-item.level-3`）。  
- **四级、五级**：继续叠在下方，**字号再小或缩进再加深**（如 `.level-4`）。  
- 保持视觉与教师端侧栏统一，避免每页新造一套侧栏样式。

## 4. 默认隐藏与展开

- 若某一级下子项**很多**，允许**默认折叠**下级。  
- 用户**点击某二级项**后，再展开其下的三级；点击三级后再展开四级，依此类推。  
- 用 `aria-expanded`、按钮或可聚焦元素，保证键盘与读屏可访问（与具体实现同步迭代）。

## 5. 右侧主内容区

- **只展示**当前在左侧**选中**的导航项所对应的内容。  
- 不要一次把「所有分类」的长说明全部铺在右侧。  
- 切换左侧选项时，更新右侧；可通过 hash 查询参数深链（例如 `#resources?tab=free`）便于分享与返回。

## 6. 适用范围

该规则适用于所有**结构复杂、有多级并列入口**的页面，包括但不限于：  

- 资料库（`#resources`）  
- 教师工作区（`#teacher` 及子路由）  
- 我的学习（`#my-learning` 等）  
- 文化、汉字学习、考试学习、会话等模块在后续迭代中增加**多子模块**时，应优先左侧层级 + 右侧面板，而不是主区平铺。

## 7. 代码与复用

- 资料页等可使用 **`ui/components/sideSectionNav.js`** 中的解析与 key 辅助（如 `parseHashSectionId`、`resourceSectionContentKeys`）。  
- 教师端已有 **`ui/pages/teacherPathNav.js`** 与 **`.teacher-shell*`** 样式，新页优先对齐。  
- 新增全站侧栏时，**优先**在 `base.css` 中扩展 **`.section-side-nav*`**，避免每页内联大段重复样式。
